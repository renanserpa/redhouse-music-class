# Diretrizes da Workspace - RedHouse Music Class

## Regras de Fluxo de Trabalho (Git & Pull Requests)

1. **Mandato de Pull Request:** NENHUMA alteração de código deve ser feita diretamente na branch principal (`master` ou `main`).
2. **Procedimento de Atualização:**
   - Criar uma branch de funcionalidade (`feat/`, `fix/`, `refactor/`).
   - Validar as alterações (lint, testes, builds).
   - Realizar o commit com mensagens semânticas.
   - Realizar o `push` para o repositório remoto.
   - Abrir um **Pull Request** detalhando as mudanças.
3. **Aprovação:** Aguardar a confirmação do usuário antes de realizar o merge ou deletar branches.

---
*Nota: Estas regras são mandatórias e sobrepõem fluxos padrão do agente.*
